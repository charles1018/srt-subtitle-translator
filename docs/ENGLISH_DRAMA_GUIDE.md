# 英語劇集翻譯模式使用指南

## 📖 概述

`english_drama` 是專為英語電視劇集、影集和電影優化的翻譯內容類型。此模式整合了經過實戰驗證的高質量翻譯提示詞，特別強化了人名處理、專業術語翻譯和台式口語表達。

## 🎯 核心特性

### 1. 人名保留規則（最關鍵特性）

**絕對規則：保留所有英文人名的原文形式，絕不音譯成中文**

#### ✅ 正確範例
- "Kylie Estevez" → 保持為 "Kylie Estevez"
- "Sylvie Brett" → 保持為 "Sylvie Brett"
- "Severide" → 保持為 "Severide"
- "Dr. Smith" → "Smith醫生" （翻譯職銜，保留人名）

#### ❌ 錯誤範例
- "Kylie Estevez" → ~~"凱莉·艾斯特維茲"~~ （禁止音譯）
- "Brett" → ~~"布雷特"~~ （禁止音譯）

#### 職銜與人名組合
當人名與職銜、稱謂結合時：
- 翻譯職銜/稱謂部分
- 保留人名的英文原文
- "Aunt Lacey" → "Lacey阿姨"
- "Lieutenant LeClerc" → "副隊長LeClerc"
- "Captain Miller" → "隊長Miller"

### 2. 地名翻譯規則

- 美國地名：保留英文，知名地點可添加中文
- 使用台灣慣用譯名

**範例：**
- "Alabama" → "阿拉巴馬"
- "Michigan" → "密西根"
- "Detroit" → "底特律"
- "House 17" → "17分局" （消防局編號）

### 3. 專業術語映射

針對特定領域（如消防、醫療）提供標準台灣術語：

| 英文 | 繁體中文（台灣） |
|------|-----------------|
| firefighter | 消防員 |
| ambulance | 救護車 |
| Truck (unit) | 雲梯隊 |
| Engine (unit) | 消防車隊 |
| Lieutenant | 副隊長 |
| Captain | 隊長 |
| shift | 班次 |
| bunk | 床位 |

### 4. 台式口語表達

使用自然的台灣國語，避免大陸用語或過於正式的表達。

| 台式表達 ✅ | 避免使用 ❌ |
|------------|-----------|
| 你還好嗎 | 你沒事吧 |
| 信我 | 相信我 |
| 怎麼那麼多人 | 為什麼這麼多人 |
| 混亂得要命 | 非常混亂 |
| 話說回來 | 但是 |
| 對 | 是的 |
| 不會吧 | 不可能吧 |

**特點：**
- 對話式、自然
- 簡潔而不生硬
- 符合台灣口語習慣
- 文化適當性

### 5. 成語與俚語翻譯

轉換為同等效果的台灣慣用語，保持原意和語氣。

**範例：**
- "knuckle sandwich time" → "我就打誰" （直接威脅，保持語氣）
- "eejits" → "白癡" （貶義但不過分）
- "barking at me" (疼痛) → "痛起來了" （具體身體感受）
- "bunking up" → "留在" （住宿/停留）

### 6. 濃縮與簡化

字幕需考慮閱讀速度，適當濃縮同時保留核心意義。

**簡化原則：**
- 移除冗餘填充詞
- 消除非必要修飾語
- 保持語意核心完整
- 優先清晰度而非字面準確性

**範例：**

原文：
```
I just hope that there's not something more serious going on
```
翻譯：
```
我只希望沒有什麼更重大的事件
```
（濃縮但保留意義）

原文：
```
he's got a great spot waiting for him up there at the FD in Detroit
```
翻譯：
```
底特律消防局有個好職位在等他
```
（精簡化，更清晰）

## 📋 使用步驟

### 在 GUI 中使用

1. **啟動應用程式**
   ```bash
   uv run srt-translator
   ```

2. **選擇內容類型**
   - 在「內容類型」下拉選單中選擇 **"english_drama"**

3. **選擇 LLM 引擎**
   - 推薦使用 OpenAI GPT-4 或 Anthropic Claude 以獲得最佳翻譯質量
   - Ollama 本地模型也支持，但可能需要較大的模型

4. **選擇字幕檔案**
   - 點擊「選擇輸入檔案」或直接拖放檔案

5. **配置選項**（可選）
   - 啟用 Netflix 風格：自動套用字符限制和格式規範
   - 調整並發請求數：根據 API 限制調整

6. **開始翻譯**
   - 點擊「開始翻譯」按鈕

### 在命令列中使用

```bash
uv run srt-translator \
  --content-type english_drama \
  --llm-type openai \
  --model gpt-4 \
  --input your_subtitle.srt \
  --output translated_subtitle.srt
```

## 🎓 翻譯範例

### 範例 1：人名保留
```
英文: Aunt Lacey, you're family.
中文: Lacey阿姨，你是家人
```
*注意：「Lacey」保持英文，「Aunt」翻譯為「阿姨」*

### 範例 2：口語表達
```
英文: What's with the crowd?
中文: 怎麼那麼多人
```
*注意：自然的台灣口語問句形式，非字面翻譯*

### 範例 3：專業術語
```
英文: They're bunking up here at 51.
中文: 他們要留在51分局
```
*注意：「House 51」翻譯為「51分局」（消防站術語）*

### 範例 4：情感語氣
```
英文: I'm gonna miss that guy.
中文: 我會想念他的
```
*注意：用「的」保留親切語氣*

### 範例 5：俚語轉換
```
英文: anybody touches my bunk, it's knuckle sandwich time.
中文: 誰敢碰我的床，我就打誰
```
*注意：「knuckle sandwich」→ 直接威脅「打誰」，保持攻擊性語氣*

### 範例 6：濃縮
```
英文: I just hope that there's not something more serious going on.
中文: 我只希望沒有什麼更重大的事件
```
*注意：精簡化同時保留擔憂/關切情緒*

### 範例 7：多行對話
```
英文:
Yeah. No doubt. But hey...
he's got a great spot waiting for him up there at the FD in Detroit.

中文:
當然了，不過話說回來
底特律消防局有個好職位在等他
```
*注意：自然流暢，濃縮，FD 擴展為消防局以求清晰*

## 🔧 進階配置

### 自訂提示詞

如果需要針對特定劇集調整術語或風格：

1. **開啟提示詞編輯器**
   - GUI 中選擇「工具」→「編輯提示詞」

2. **選擇 english_drama 類型**
   - 在內容類型下拉選單選擇 "english_drama"

3. **修改提示詞**
   - 可以添加劇集特定的術語映射
   - 調整語氣或風格指導
   - 添加角色名稱列表

4. **儲存並測試**
   - 儲存修改後的提示詞
   - 使用小範圍字幕測試效果

### 術語表自訂

針對特定劇集領域（如法律劇、醫療劇），可以在提示詞中添加領域專用術語：

**範例（醫療劇）：**
```
- "ICU" → "加護病房"
- "resident" → "住院醫師"
- "rounds" → "查房"
- "code blue" → "緊急救援"
```

**範例（法律劇）：**
```
- "attorney" → "律師"
- "district attorney" → "檢察官"
- "arraignment" → "提訊"
- "plea bargain" → "認罪協商"
```

## 💡 最佳實踐

1. **首次使用建議**
   - 先用小檔案（10-20 行字幕）測試
   - 檢查人名是否正確保留
   - 確認術語翻譯符合劇集背景

2. **質量檢查要點**
   - ✅ 所有英文人名保持原文
   - ✅ 使用台灣慣用表達（非大陸用語）
   - ✅ 專業術語準確
   - ✅ 字符數符合限制（若啟用 Netflix 風格）
   - ✅ 情感語氣保留

3. **效能優化**
   - 使用 OpenAI GPT-4 或 Anthropic Claude 3.5 Sonnet 獲得最佳質量
   - 啟用翻譯快取以節省 API 成本
   - 合理設定並發數（建議 3-5）

4. **常見問題排查**
   - **人名被音譯**：檢查是否正確選擇了 english_drama 模式
   - **術語翻譯不準確**：可自訂提示詞添加特定術語表
   - **語氣過於正式**：確認台式口語表達模組已啟用
